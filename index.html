Hộp thông tin phân cấp (Callouts)

Thông tin (Info): Cổng kết nối Gateway chạy mặc định ở port 8080.
Thành công (Success): API trả về mã 200 OK khi cập nhật hồ sơ thành công.
Cảnh báo (Warning): Token truy cập (Access Token) sẽ hết hạn sau 15 phút.
Nguy hiểm (Danger): Xóa nhóm chat sẽ xóa toàn bộ dữ liệu tin nhắn vĩnh viễn.

# Quy tắc Đặc tả & AI Ruleset

Quy chuẩn thiết kế đặc tả hệ thống và bộ quy tắc bắt buộc dành cho lập trình viên và các trình sinh mã tự động (AI Code Generators).

Bộ Quy Tắc Đặc Tả Kỹ Thuật (AI Ruleset Documented)

Nguyên tắc bắt buộc: Khi yêu cầu AI sinh mã HTML cho một trang đặc tả chức năng mới, luôn yêu cầu AI đọc và tuân thủ tuyệt đối các quy định dưới đây.

1. Quy tắc Viết Mã

  • Chỉ sử dụng các class CSS đã được định nghĩa sẵn trong design system này.
  • Tuyệt đối không thêm thuộc tính style="" inline trong các thẻ HTML.
  • Không tự ý tạo thêm class mới ngoài hệ thống thiết kế.
  • Chỉ trả về đoạn mã HTML Fragment (khối nội dung), không bao gồm các thẻ <html>, <head>, <body> trừ khi được yêu cầu tạo tệp độc lập.

2. Cấu trúc đặc tả Chức năng

Mọi tài liệu mô tả API/Chức năng phải được trình bày theo đúng trình tự:

  1. Thông tin chung: Badge (HTTP method) + Status Label + Kicker (Tên chức năng) đặt cùng hàng.
  2. Mô tả nghiệp vụ: Tóm tắt nghiệp vụ trong 2-3 câu bằng thẻ <p>.
  3. Sơ đồ luồng: Khung .diagram-wrapper chứa <pre class="mermaid"> và chú thích .diagram-caption.
  4. Bảng tham số I/O: Bảng .table-tech liệt kê các tham số (Tham số bắt buộc có <span class="required-star">*</span>).
  5. Ngoại lệ & Giới hạn: Sử dụng .callout.warning hoặc .callout.danger.
  6. Ví dụ dữ liệu: Khối mã nguồn .code-wrapper chứa Request/Response JSON thực tế.

Quy tắc chọn Sơ đồ (Diagram Mapping Rules)

Dạng Sơ đồ Loại Mermaid Mục đích sử dụng
Use Case graph TB hoặc graph LR Biểu diễn tác nhân và các chức năng hệ thống cung cấp.
Kiến trúc graph TD Phân tầng kỹ thuật (Presentation, Gateway, Microservices, Storage).
Triển khai graph LR hoặc graph TD Mô tả hạ tầng vật lý, mạng, VPC và các tài nguyên Cloud.
Trình tự sequenceDiagram Mô tả luồng giao tiếp thời gian thực, có số thứ tự tự động.
Hoạt động flowchart TD hoặc flowchart LR Biểu diễn luồng logic nghiệp vụ, các bước rẽ nhánh điều kiện.

Quy chuẩn Thiết kế Sơ đồ Mermaid

  • Tối giản giao cắt: Định hướng luồng rõ ràng (TD: Top-Down hoặc LR: Left-Right) để giảm thiểu đường chỉ chéo nhau gây rối mắt.
  • Nhãn rõ ràng: Luôn đặt tên rõ ràng cho tác nhân/thành phần, ví dụ sử dụng actor User as Client User hoặc participant GW as API Gateway.
  • Màu sắc tối giản: Không tự ý thay đổi màu lòe loẹt. Dùng config base-theme trùng với gam màu kem ấm của hệ thống.
  • Không tự co giãn méo mó: Cấu hình useMaxWidth: false để sơ đồ hiển thị kích thước gốc sắc nét và có thể cuộn ngang khi cần thiết trên di động.
Tip: Định nghĩa class CSS trong Mermaid (classDef) để gom nhóm màu sắc Client/Service/DB.
